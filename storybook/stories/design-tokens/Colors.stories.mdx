## Global Colors

Generic Global colors to use within a Mastercard application.

These tokens are inherited by all other token types e.g. `made-color-orange-04` is the value of the `made-color-brand-primary` tokens.

They can be directly used if there are no role based token to meet your requirements,
but use sparingly as these will tokens are not themeable e.g.
`made-color-orange-04` should always be orange, use `made-color-brand-primary` or the token that best describes your use case.

## Role-based tokens

By bringing semantic meaning to tokens with roles, developers don't have to second-guess what color to apply for a background. They don't have to choose between gray.01 or white. Instead, they'd apply a design decision with an example name of made-color-background-default. 
By using this approach, changing from light to dark mode is only a matter of switching the values this color has, instead of overriding the existing CSS.

## ⚠️ Deprecated Tokens

In order to improve consistency across the library and reduce the number of tokens required to create a new theme we are deprecating the tokens below.

Please replace deprecated tokens with the new tokens below.

If you need to continue using these tokens, you can import a deprecated file in the format that you need e.g.

```
<link rel="stylesheet" href="[path-to-directory]/deprecated.css" />
```

# Q2 (July) 2022

Made. July release for design tokens and CSS includes:

- <a onClick={linkTo("design-tokens-theming--color")}>Themeable tokens</a> that allow
  teams to re-theme applications in a matter of minutes.
- Additional <a onClick={linkTo("utility-classes-accessibility--accessibility")}>utility classes</a> to give developers more flexibility to modify and extend existing components.
- Feature requests from discussions, issues and improvements highlighted by the Made. community.

## Theming üé®

The Made. Framework is evolving our design tokens to scale across products by support theming and white-labelling.

We have flagged 40 color, 25 typography, 5 border radius and 6 grid spacing themeable tokens that will cascade to other design tokens rather than teams having to change each token separately.
This allows teams to re-theme applications in a matter of minutes.
You may want to change only a small number of tokens to re-theme primary colours, typography or border radius or you might need to change more of the these tokens to create a dark theme for example.

In the process of defining these tokens we have created consistent patterns for disabled, hover and selected interactive states,
replaced the 46 button tokens to 12 action colours and added additional tokens for secondary brand colours that will cascade to
secondary buttons for example.

We have added design tokens that will allow teams to re-theme
all font families and font weights for headings. We have added border radius tokens to theme the border radius of UI components across your application,
re-theming from round to square buttons, cards, pills, form elements.

See the new <a onClick={linkTo("design-tokens-theming--color")}>Design Tokens Theming</a> Guide for more details on theming.

### Interactive states

In the previous versions of the Made. B2B library there were a number of styles for disabled states, hover and selected styles depending on context.
Made. has defined styles for the states below that should be applied consistently across components in order to increase usability.
